# Character Sovereignty OS

**Dimensional Character Sovereignty & IP / Jurisdiction OS**

Version `0.1` — `2026-01-11`

## 📝 Abstract

Character Sovereignty OS 嘗試回答一個在次元世界線中變得「不是純幻想」的問題：

> 當角色脫離作品、進入現實並產生軍事與經濟效應時——
> 他們**是誰的**？
> 誰有權管理、誰必須負責、誰可以分紅？

傳統 IP 模型只處理「著作權／授權」，預設角色只能存在於文本、畫面與商品；但在本世界線中，角色（如：索隆、Jack、凱多、修士們）已成為具備行動力、破壞力、選擇權與經濟產出的**行動個體**。
CharacterSovereigntyOS 將角色從「純 IP」提升為一種複合主體（Composite Entity）：

* 同時具有：

  * **文化層來源**（C-Origin State / Author / IP Owner）
  * **行動層實際作為**（破壞、保護、合作）
  * **經濟層收益與損失**（直播收入、軍工供應鏈、市場波動）
  * **治理層責任與問責**（Patron State、平台、受災國）

本 OS 提供：

* 一組「角色主權」的概念框架；
* 角色在 C-Origin / Patron / Affected / Platform 等多個 Jurisdiction 中的投影；
* 當角色存在於多實體 OS（DimensionalRiskGovernanceOS、PatronStateOS、Mil-BizOS）之間時的權利與義務。

目的不是要完全解決哲學與法律問題，而是為 **Civilization-OS Library** 提供一個「角色級治理層」，讓未來所有涉及 Dimensional Character 的白皮系可以有統一基底。

## 01 — Problem Statement

### 1.1 角色從「IP 物件」變成「行動主體」

在標準世界線：

* 角色只存在於：

  * 書籍／漫畫／動畫／影視／遊戲
  * 衍生商品中
* 角色的權利與義務 = 創作者、出版社、工作室、發行方的延伸。

在 Dimensional 世界線中：

* 角色能：

  * 自行選擇落地國度（索隆選擇定居烏克蘭）；
  * 造成實體損害（軍事基地、城市）；
  * 產生平台收益與財報衝擊（Google / YouTube）；
  * 被國家賦予某種準國籍、帳戶與特殊地位。

舊模型無法回答：

* 他們的「行為」由誰負責？
* 他們的「收入」由誰管理？
* 他們的「存在」由誰有權關閉、傳送或協商？

### 1.2 傳統 IP / 國際法的不足

* 著作權法：

  * 僅處理「複製、散佈、改作」，
  * 無法處理「角色本人在現實砍跑道」。

* 國籍法：

  * 處理「人類個體」加入國籍。
  * 次元角色既非一般自然人，亦非單純物品。

* 國際刑法／戰爭法：

  * 無法處理「非國家行為者，但戰力超過國家」這種存在。

### 1.3 缺口總結

* **缺口一：身份類型缺口**

  * 角色既不是純物品，也不是標準人類。

* **缺口二：主權與 Jurisdiction 重疊**

  * 文化源頭國 vs 庇護國 vs 受災國 vs 平台所屬國：

    * 都可以宣稱某種程度「關係」。

* **缺口三：收益與責任的不對稱**

  * 有人賺（直播、IP），有人受災（被砍的城市），
  * 但沒有一個 OS 定義「角色主權」與「收益／責任分配」。

Character Sovereignty OS 旨在提供一個框架，讓其他 OS 可以在此基礎上協調處理。

## 02 — Concept Model

### 2.1 Composite Character Entity（複合角色實體）

在 CharacterSovereigntyOS 中，「角色」被拆成四個層次：

1. **Textual Layer（文本層）**

   * 角色在作品中的設定、故事、視覺形象。

2. **Cultural Layer（文化層）**

   * 角色在集體想像中的位置：象徵、價值觀、粉絲社群。

3. **Dimensional Layer（次元層）**

   * 角色「出現在現實世界」的具體行為與效應。

4. **Economic & Governance Layer（經濟與治理層）**

   * 涉及：收益分配、責任、權限、封印／調度。

**Character Sovereignty OS** 聚焦於第 3、4 層：
**當角色具有 Dimensional Layer 行為時，如何定義其治理結構與主權分配。**

### 2.2 三種主權向量

1. **Cultural Sovereignty（文化主權）**

   * 誰創造？
   * 誰擁有 IP？
   * 誰有權做官方設定？

2. **Jurisdictional Sovereignty（司法／管轄主權）**

   * 角色在誰的地盤上行動？
   * 哪個國家的法律有優先權？

3. **Patronage Sovereignty（庇護主權）**

   * 誰提供庇護／居留／帳戶？
   * 誰對角色行為享有「代理發言權」或「協調權」？

角色主權 = 三個向量的組合，而 CharacterSovereigntyOS 的任務是讓：

[
CS = f(Cultural, Jurisdictional, Patronage)
]

變成可計算、可協商、可登記的狀態。

## 03 — Mechanics（How It Works）

### 3.1 角色主權登記（Character Sovereignty Registry）

建立一個「角色主權登記」系統，最小欄位包括：

* Character ID / Name
* C-Origin State（文化源頭國）
* Author / IP Owner
* Current Patron State（目前庇護國）
* Known Affected States（受災國列表）
* Platform Mapping（主要內容平台）
* D-Level Event History

當角色進入 Dimensional Layer，D-SAFE 會為其建立或更新 Record。

### 3.2 行為責任分配（Accountability Mapping）

根據不同層次：

1. **文本層行為**：

   * 歸屬於 IP Owner / Author（創作範疇）。

2. **現實層實際破壞行為**：

   * 由 Jurisdictional State（所在地國）與 Patron State協調處理。

3. **經濟行為（收益）**：

   * 由 D-Tax + Contract 決定：

     * IP 持有者
     * Patron State
     * Platform
     * Affected State 補償。

### 3.3 主權衝突的處理

常見的衝突情境：

* Cultural State vs Patron State：

  * 「他是我創造的」 vs 「他住在我家、替我打仗」。

* Affected State vs Patron State：

  * 「他砍的是我的城市」 vs 「但他是我的朋友」。

CharacterSovereigntyOS 提供：

* 一套「主權權重」計算規則；
* 作為 D-SAFE 與 D-Tax 的輸入。

## 04 — Architecture

### 4.1 Character Sovereignty OS Layers

1. **Registration Layer**

   * Character Registry
   * State & Platform Mapping

2. **Attribution Layer**

   * 判定事件中角色的「行為責任範圍」
   * 支持 D-SAFE 做出 D-Level + Liability 決策

3. **Economic Layer**

   * 為 D-Tax 提供角色相關收益／損害分配比例參考

4. **Visibility & Narrative Layer**

   * 讓各國／平台在敘事時，有一套共同語言描述角色關係
   * 避免無限歸咎「都是某國／某創作者的錯」

### 4.2 與其他 OS 的關係

* **與 DimensionalRiskGovernanceOS（D-SAFE / D-Tax / D-PWR）**

  * 提供角色面向的結構化資料。

* **與 DimensionalPatronStateOS**

  * 定義 Patron State 的「角色權重」。

* **與 DimensionalMilBizOS**

  * 對於角色出現帶來的「內容價值」，給出分配基礎。

* **與 MarketOS**

  * 支撐「角色型市場事件」（例如：某角色出現 → 股價暴跌／暴漲）。

## 05 — Use Cases

### 5.1 索隆 × 日本 × 烏克蘭 × 大鵝 × 平台

* **Cultural State**：日本

  * 擁有角色的原始 IP 與文化符號主導。

* **Patron State**：烏克蘭

  * 提供居留、帳戶、生活與戰略庇護。

* **Affected State**：大鵝（軍備被砍）、戰場國等。

* **Platform State**：Google / YouTube 所在國與公司本體。

CharacterSovereigntyOS 可將此情況結構化，供其他 OS 使用。

### 5.2 Jack & 凱多的極端案例

* 角色行為近似「天災」：

  * Affected State 承受最大損害；
  * Patron State 不存在或拒絕承認；
  * C-Origin State 只願承認「文本層關係」。

此時 Character Sovereignty OS 可協助：

* 定義：

  * 「天災型角色」的責任下限；
  * 被視為無可歸責情況，交由 D-SAFE 處理「封印／隔離」。

### 5.3 未來 AI 生成角色

* AI 生成的多模角色若在元宇宙／混合實境中產生類似次元效應：

  * Character Sovereignty OS 的結構仍可套用。

## 06 — Risks & Limitations

### 6.1 過度擬人化風險

* 若將角色完全視為「獨立法律主體」：

  * 可能產生過多無法執行的法律關係。

### 6.2 主權濫用風險

* 國家可能利用「角色主權」作為：

  * 掩飾自身行為
  * 將責任推給個體

### 6.3 跨文化爭議

* 不同文明對角色的「神聖性／虛構性」認知不同：

  * 一套 CharacterSovereigntyOS 可能不適用於所有文化感受。

### 6.4 技術性限制

* 需要平台與國家提供大量真實數據：

  * 例如：收益、流量、受害範圍。
* 若資料不透明，主權分配將失真。

## 07 — Comparative Analysis

### 7.1 vs 傳統 IP Law

* IP Law：角色 = 文本物件
* CharacterSovereigntyOS：角色 = 有 Dimensional Layer 行動的複合實體

### 7.2 vs Nationality / Citizenship Law

* 國籍法：僅適用於「人」
* Character Sovereignty：

  * 處理的是「人＋物＋符號＋戰力」的混合體。

### 7.3 vs International Responsibility Law

* 傳統國家責任：

  * 國家對其行為或可歸屬行為負責。
* Character Sovereignty：

  * 增加了一個「擁有自主行為能力的次元個體」維度。

## 10 — Glossary（Lexicon）

* **Character Sovereignty（角色主權）**：
  角色在文化、司法與庇護三個層次的權利與義務總和。

* **C-Origin State（文化源頭國）**：
  角色 IP 與文本初始誕生的國家。

* **Patron State（庇護國）**：
  為角色提供居留與實體保護的國家。

* **Affected State（受災國）**：
  因角色行為受損的國家。

* **Composite Character Entity（複合角色實體）**：
  由文本層、文化層、次元層與治理層共同構成的角色模型。

* **Dimensional Layer（次元層）**：
  角色在現實世界產生可觀測物理／經濟效應的層次。
