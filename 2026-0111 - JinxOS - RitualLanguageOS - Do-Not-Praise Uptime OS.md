## 📝 Abstract

**Do-Not-Praise Uptime OS** 描述全球所有 IT／機房／SRE／DevOps 團隊共同的語言禁忌：

> **「最近 uptime 都 100% 呢！」**
> **「這台伺服器超穩。」**
> **「這週都沒爆過。」**

這些語句被視為「召喚故障」的高危語言，
會引起團隊激烈反應：

* 「不要亂講！」
* 「你完了。」
* 「你講完等一下就爆了。」

本 OS 將此現象抽象為：

1. **語言觸發的風險反射（Risk Reflex Trigger）**
2. **防自滿防線（Anti-Complacency Wall）**
3. **集體注意力同步機制（SRE Awareness Sync）**
4. **歷史災害的敘事殘影（Folklore Memory Binding）**
5. **運作行為強化（Operational Guarding）**

這篇是 JinxOS 世界線中最「工程師特色」的語言文明。

## 01 — Problem Statement

### 1.1 為什麼「稱讚穩定」會被視為危險？

SRE／IT 團隊的核心痛點：

* 系統穩時 → 容易鬆懈
* 故障通常在注意力下降時發生
* 任何過度自信都可能造成忽略性錯誤
* 風險是長期存在的，而非語言觸發

但語言會讓人誤以為：

> **「既然很穩 → 可以放鬆一下。」**

而這種放鬆 = 風險。

因此形成：

* 自然禁止
* 文化強化
* 故事綁定
* 團隊反射

### 1.2 語言禁忌是一種「高壓認知防護系統」

IT 系統有「隨時爆炸」的特性。
稱讚 uptime 會被視為「挑釁命運」。

JinxOS 將這當作一種文明策略：
**語言反向儀式（Inverse RitualSpeech）。**

## 02 — Concept Model

### 2.1 Do-Not-Praise Uptime OS =

**禁語 × 認知防線 × 過度自信抑制 × 風險敘事 × 行為同步**

核心組件：

1. **Trigger Word（觸發語）**

   * “穩”
   * “今天都沒爆”
   * “100% uptime”
   * “看吧，其實很穩”

2. **Group Reflex（團隊反射）**

   * 「不要講！」
   * 「你害的喔！」
   * 「你講完一定出事！」

3. **Attention Reset（注意力重啟）**
   語言會強迫團隊回復警覺模式。

4. **Risk-Folklore（風險故事綁定）**

   * 曾經有人稱讚穩 → 5 分鐘後大爆炸
   * 這類故事大量流傳，成為 SRE 實務教育的一部分

5. **Operational Guarding（行為防護）**

   * 讚 uptime 後 → 會多做一次 check
   * 或默默做好 rollback 準備

### 3.1 語言禁忌事件流（Uptime Jinx Flow）

```
[Engineer says "很穩/沒爆"]
          ↓
[Team Shock Reaction]
          ↓
[Shared Anxiety rises]
          ↓
[Attention Reset / Risk Scan begins]
          ↓

          ↓

```

### 3.2 認知心理學機制

* **可得性偏誤 Availability Bias**
  稱讚後爆炸的案例最容易記住。

* **風險預期敏感性 Threat Anticipation**
  高壓團隊天然敏感。

* **負面強化 Negative Reinforcement**
  每次稱讚都伴隨緊張 → 強化禁語。

* **群體同步 Collective Synchronization**
  語言觸發整個團隊的注意力重新 align。

## 05 — Use Cases

### 5.1 IT / SRE 團隊

* 稱讚 uptime → 團隊瞬間反射：
  「閉嘴！」
* 接著開始全民 double-check。

### 5.2 DevOps

* CI/CD pipeline 特別容易受到語言觸發
* “這次應該很快” = 大忌

### 5.3 機房工程

* 語言禁忌降低維運鬆懈
* 不說「穩」反而更穩

### 5.4 軍武 / 雷達站

* 不能說「最近都沒狀況」
* 語言禁忌 → 認知警戒維持

## 06 — Risks & Limitations

* 過度禁語可能限制健康溝通
* 新人可能不理解禁語 → 文化隔閡
* 禁語不能替代正式監控系統
* 若將禁語誤解成「魔法」會偏離 OS 目的
